# Antigravity Workflows

This directory contains executable, robust workflows designed for standardizing agent operations. They dictate exactly **how** an agent should execute a complex process from start to finish.

Unlike rules (which are passive constraints) and skills (which are domain knowledge), workflows are **narrative scripts** detailing a step-by-step strategy. Below are the workflows available in this kit.

## 🛠️ Build & Dev Workflows

- **`plan.md`**: Architectural design and project tracking scaffolding. Master artifact generation guidelines.
- **`create.md`**: Rapid scaffolding of clean code skeletons using best practices.
- **`ui-ux-pro-max.md`**: A **11,000+ byte** masterpiece of a frontend design workflow specifying pixel-perfect execution, margin rules, interactions, and aesthetic audits.
- **`brainstorm.md`**: Strategic analysis and feature ideation loops.
- **`enhance.md`**: Methodological approach to refactoring, performance profiling, and code cleanup.

## 🐛 Debugging & Quality

- **`debug.md`**: Root-cause analysis protocols. Enforces adding temporary logging and isolated testing before modifying underlying code.
- **`test.md`**: Standardized structure for unit and integration testing workflows.
- **`preview.md`**: Post-change compilation and manual/visual reviews.

## 🚀 DevOps & Project Management

- **`deploy.md`**: Secure checks, asset minification, routing audits, and CI/CD preparation rules.
- **`orchestrate.md`**: Extremely advanced command script to choreograph multiple AI agents passing tasks back and forth.
- **`status.md`**: Status reporting generation to track overall execution loops.

## 📝 OpenSpec Architecture (study8677)

A modular architectural-proposals system ensuring multi-developer codebases scale safely.

- **`openspec-proposal.md`**: Scaffold a new OpenSpec change. Write design documents and tasks without writing actual implementation code.
- **`openspec-apply.md`**: Validates proposal files, executes strictly bounded tasks, and verifies changes exactly match spec configurations.
- **`openspec-archive.md`**: Marks successfully applied specs as historical archives to keep active working directory clean.

> **Agent Note**: If the user asks for a feature, process, or workflow that matches any file in here, ALWAYS use `view_file` to read the corresponding `.md` file first and strictly follow its steps.
